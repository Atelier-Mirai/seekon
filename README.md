ソースコードの共用について
-----------------------------------------------------------------------------
一般的には、Git(ギット)と呼ばれるバージョン管理システムを使って管理します。
ウェブサイトを作成する祭には、毎日、いろいろなコードを書き加えて行きます。
昨日の時点に戻したいな〜 であったり、
お試しで少しこの部分を変更してみたいであったり、
バージョン管理システムを使っていると、直ぐに出来るので習得してみてください。
コマンドを入力して行うのが基本ですが、
初心者でも容易に出来るよう、SourceTree(ソースツリー)というソフトも提供されています。

また、ウェブサイトは自分一人で作成することもありますし、複数人で協力して作成することもあります。
ソースコードのバックアップや、共用に便利なサイトが、GitHub(ギットハブ)です。

[わかばちゃんと学ぶ Git使い方入門](https://www.amazon.co.jp/dp/4863543433) が分かりやすく解説してあります。

画像について
-----------------------------------------------------------------------------
写真用の画像形式として、従来は.jpg(じぇいぺぐ)が用いられてきました。
そして、次世代画像形式として、WebP(うぇっぴー)が登場しています。
見た目は変わらず、ファイルサイズを小さくすることが出来る画像形式です。
[次世代画像形式のWebP](https://ics.media/entry/201001/)参照  

また、2kや4kなどの大きな画像を載せても、
iPhoneなどで閲覧するには、無駄に高解像度なだけで、ファイルサイズが大きい分、速度面でも不利になりますので、
適当な大きさに縮小されている画像ファイルで十分です。

ファビコン / アップルタッチアイコンの指定について
-----------------------------------------------------------------------------
ファビコンとは、ブラウザのタブの左側に表示される小さなアイコン（マーク）のことです。
アップルタッチアイコンは、ウェブサイトを「ホーム画面に追加」する際に使われるアイコンです。
一般的には会社のロゴマークなどを表示します。
ファビコンジェネレータ https://ao-system.net/favicongenerator/ で作成することが出来ます。
ロゴマークになりそうな画像として、images/logo.png を基に設定してみました。

Modaal について
-----------------------------------------------------------------------------
モーダルウィンドウを実現するライブラリは多数ありますが、その一つが Modaal です。
Modaal は、専用のスタイルシートが必要ですので、head 内に ModaalのCSSを読み込みます。
Modaal は、「jQuery(じぇいくえりー)」という、JavaScript を簡単に書く為のライブラリを前提としていますので、jQuery を body の最後に読み込みます。
そして、jQueryを読み込んだ後に、Modaal の JavaScript を読み込みます。
それから、body の最後に Modaalを設定(カスタマイズ)するためのJavaScript(modaal-custom.js)を読み込みます。
modaal-custom.js内には Modaalを設定するためのJavaScriptが書かれています。
```
$(".inline").modaal({
});
```
がその部分です。
「$」は、jQuery の関数です。htmlから、「.inline」と、inline クラス名が付けられた要素を抽出し、modaal メソッドを実行することで、「.inline」要素が、モーダル動作をするようになります。

aタグの id属性はtraining, class名はinlineにします。
<a href="#training" class="inline">

すると、<a href="#training"> がクリックされたときに、<div id="training"> が表示されます。
<div id="training" class="hidden"> と、「hidden」クラスを付与しておくことで、クリックされていないときには、表示されないようにできます。

.hidden の css は、stylesheets/utility.css に、次のように書いています。
```
.hidden {
  display: none;
}
```

display: none; とありますので、hiddenクラスは、非表示になります。

html, css, javascript の関係について
-----------------------------------------------------------------------------
htmlは、文書構造、表示したい内容を記述します。
cssは、html要素の装飾や装飾を担当します。
JavaScriptは、利用者の操作に応じてウェブサイトに動きを与える、制御する為に使われるプログラミング言語です。
例えば、
ある特定のリンクがクリックされたときに、ある特定の要素が表示されるようにする（モーダルウィンドウの例）
ある特定のリンクがクリックされたときに、表示される位置を変更する（先頭に戻るボタンの例）です。
ある特定の写真をクリックしたときに、それを大きく表示する（写真ギャラリー Magnific Popupの例）です。

html では、リンクであることを示す為に、<a href="#training" class="inline gold button"><h3>TRAINING QUALITY</h3></a>と書きます。
そして、飾り付けを行う為に、「gold button」クラスを付与します。

実際にどのような飾り付けを行うかは、cssで記述します。
飾り付け内容に関する記述は stylesheets/button.css に書かれており、
.gold.button { 金色のボタンの形に彩る } となっています。
このように、<a>タグ はたくさんあるので、飾り付けを行いたいタグには、クラス名を付けて、
飾り付けの内容は、それぞれのクラス名に応じて、css でいろいろ記述するのが一般的です。
クラス名は、自分自身で分かりやすい名前を付けることが出来ますので、
<a class="kiniro-no-botan"> のようにしたり、
<a class="sentou-ni-modoru-botan"> というようにすることも出来ます。

JavaScript は、
ある特定のリンクがクリックされたときに、表示される位置を変更する（先頭に戻るボタンの例）などのように使います。
このとき、html の要素には、<a class="inline"></a> や、<p id="page-top"></p>のように、クラス名やID属性を付与しておき、JavaScriptのプログラムコードには、「inline」クラスがクリックされたときにはこういう処理をするであったり、「page-top」がクリックされたときには、こういう処理をするようにプログラムを書いていきます。
また、直接プログラムを書くと長くなりますし、他のウェブサイトを作るときにも使い回し出来るよう、別のファイルにプログラムコードを書いておき、実際に使うhtml内に読み込んで使う方法が普通です。
先頭に戻るプログラムの例でしたら、bodyタグの最後に、<script src="javascripts/page-top.js"></script> と書き、javascriptsディレクトリ内にある、「page-top.js」というファイルを読み込んで使っています。

css ファイルについて
-----------------------------------------------------------------------------
一つの巨大な css ファイルを作ると、どこに何が書かれているのか管理が大変になってきます。
その為、部品ごと、機能ごとに css ファイルを分割して、
使うときに必要な css ファイルを読み込むようにすると良いです。

```
<!-- 自分で作ったスタイルシート -->
<link rel="stylesheet" href="stylesheets/utility.css">
<link rel="stylesheet" href="stylesheets/button.css">
<link rel="stylesheet" href="stylesheets/text.css">
(略)
<link rel="stylesheet" href="stylesheets/style.css">
```

レスポンシブレイアウト / ブレークポイントについて
-----------------------------------------------------------------------------
iPhoneでウェブサイトを見る人が大半ですので、
「モバイルファースト」として、まずは iPhone での見た目を整える。
て、ウェブサイトを閲覧するデバイス（iPhone, iPad, Macなど）の幅に応じて、
480px 768px 960px 1280px などで区切りを設けて、
レイアウトやデザインの微修正を行うのが一般的な作り方で、レスポンシブレイアウトと呼ばれます。
ブレークポイントをどのように設定するかは、いろいろ考え方はありますが、
一般的なブレークポイントは、480px 768px 960px 1280px などが良く使われています。
ブレークポイントを一つ設定するなら、iPadの横幅となる 768px で区切るのが一般的でしょう。

ウェブサイトの作り方について
-----------------------------------------------------------------------------
（最近では一ページのみで完結するデザインのサイトもありますが）一般的にウェブサイトは、多くのページから構成されます。このとき、トップページと、それ以外のページ（下層ページ）では、少しデザインが異なりますので、
トップページには、<body class="top">、下層ページには、<body class="site">などとクラス名を付けて区別する方法があります。

そして、一般的には、ウェブサイトは次のようになります。
```
<!DOCTYPE html>
<html>
  <head>
  </head>
  <body class="top"> または <body class="site">
    <header></header>
    <nav></nav>
    <!-- 内容 -->
    <footer></footer>
  </body>
</html>
```

ですので、トップページの完成後、それをコピーして、下層ページを作っていくと、共通する<head>や<header><nav>などをそのまま使えるので作成が楽でしょう。

また、css では、
.top h1 { トップページ用の装飾いろいろ }
.site h1 { 下層ページ用の装飾いろいろ }
のように、クラス名を付けて記述していくとそれぞれのページにスタイルを提供することが出来ます。

トップページと、下層ページ共通に適用させたい場合には、
header { 装飾いろいろ }
footer { 装飾いろいろ }
とクラス名を付けずに記述します。

さらに、静的サイトジェネレータである「middleman」（ミドルマン）などを用いると、
共通する部品を纏めて管理することが出来るので大変便利です。

Fomantic UI について
-----------------------------------------------------------------------------
[Fomantic UI(ふぉまんてぃっくゆーあい)](https://fomantic-ui.com/)は、CSS フレームワークと呼ばれるものです。
ボタンやテーブル、フォームなどのデザインを一から行っていくのはなかなか手間暇かかるものです。
そういったときに先人達が作成してくれたCSSを用いるのは、
労力も節約でき、手軽に洗練されたデザインを適用することが出来るので、お薦めです。

[ボタンの例](https://fomantic-ui.com/elements/button.html)
<button class="ui red button">Red</button> と書くことで、赤いボタンを作成できます。
